**1b. Add batch metric rank tracking to train_epoch in part1/train.py:**

Modify the `train_epoch` function to track running percentile/rank of each batch's loss and gradient norm relative to history in the current epoch. This is computationally negligible (scalar comparisons on Python lists).

At the top of `train_epoch`, before the for loop, add:
```python
    # Batch metric rank tracking (per-epoch running history)
    loss_history = []
    grad_norm_history = []
```

Inside the for loop, after `grad_norm` is computed (after `clip_grad_norm_`) and before the existing `log_epoch_metrics` call, add:
```python
        # Track rank of current batch metrics relative to epoch history
        import bisect
        loss_val = loss.item()
        bisect.insort(loss_history, loss_val)
        loss_rank = loss_history.index(loss_val) / max(len(loss_history) - 1, 1)
        # loss_rank: 0.0 = lowest loss this epoch, 1.0 = highest

        bisect.insort(grad_norm_history, grad_norm)
        grad_norm_rank = grad_norm_history.index(grad_norm) / max(len(grad_norm_history) - 1, 1)
```

Then add the rank metrics to the existing `log_epoch_metrics` call for batch metrics. Update the existing call from:
```python
        log_epoch_metrics({
            "batch/loss": loss.item(),
            "batch/gradient_norm": grad_norm,
            "batch/lr": optimizer.param_groups[0]["lr"],
        }, step=global_step)
```
to:
```python
        log_epoch_metrics({
            "batch/loss": loss.item(),
            "batch/gradient_norm": grad_norm,
            "batch/lr": optimizer.param_groups[0]["lr"],
            "batch/loss_rank": loss_rank,
            "batch/gradient_norm_rank": grad_norm_rank,
        }, step=global_step)
```

Move the `import bisect` to the top of the file (with other imports) rather than inside the loop.

**Important:** Do NOT modify any other functions. The rank tracking is epoch-scoped (lists reset each epoch via new list creation at the top of train_epoch).
