**Part A: Modify `part1/dpo_loss.py` -- add per_token parameter to `compute_restricted_log_probs`.**

Add an optional `per_token` parameter (default `False` to preserve backward compatibility). When `per_token=True`, return `(per_token_log_probs, mask)` tuple of shape `(B, T)` instead of the summed `(B,)` tensor. This is needed because true CISPO clips per-token IS ratios per user decision ("per-token policy ratio, true to CISPO paper formulation").

The change is minimal -- the function already computes `per_token` and `mask` internally. Just add a branch before the `.sum()`:

```python
def compute_restricted_log_probs(model, encoder_input, encoder_mask,
                                  decoder_input, decoder_targets, pad_idx=0,
                                  per_token=False):
    # ... existing code up to per_token and mask computation ...
    if per_token:
        return per_token, mask  # (B, T), (B, T)
    seq_log_probs = (per_token * mask).sum(dim=-1)
    return seq_log_probs
```

**Part B: Create `part1/rl_loss.py` with all RL loss functions.**

This file implements the mathematical core. Create these functions:

1. **`grpo_loss(current_log_probs, old_log_probs, advantages, epsilon=0.2)`**
   - Sequence-level GRPO with PPO-style min(ratio*A, clip(ratio)*A)
   - Input: (N,) tensors, Output: scalar loss + diagnostics dict
   - Diagnostics: `clip_frac`, `mean_ratio`
   - Follow the exact formula from the research doc's "GRPO Loss Function" section

2. **`cispo_loss(current_log_probs, old_log_probs, advantages, epsilon_high=0.3)`**
   - Sequence-level CISPO: `clamp(ratio, max=1+eps_high).detach() * advantage * log_prob`
   - The `.detach()` on the clamped ratio is CRITICAL -- this is the core CISPO innovation
   - Diagnostics: `clip_frac`, `mean_ratio`

3. **`cispo_loss_per_token(per_token_logps, old_per_token_logps, advantages, completion_mask, epsilon_high=0.3)`**
   - Per-token CISPO matching MiniMax-M1 paper Equation 4
   - Input: (B*G, T) per-token log probs, (B*G,) advantages, (B*G, T) mask
   - Normalize by total valid tokens: `masked_loss.sum() / total_tokens`
   - Broadcast advantages to token dimension with `.unsqueeze(1)`
   - Diagnostics: `clip_frac`, `mean_ratio`
   - This is the implementation requested by user decision ("per-token policy ratio, true to CISPO paper formulation")

4. **`compute_group_advantages(rewards, group_size, use_std=True)`**
   - Group-relative normalization: reshape (B*G,) to (B, G), compute per-group mean/std
   - When `use_std=True`: `(rewards - mean) / clamp(std, min=1e-8)` (standard GRPO)
   - When `use_std=False`: `(rewards - mean)` only (Dr. GRPO variant)
   - Return `(advantages, zero_std_frac)` where zero_std_frac = fraction of groups with std < 1e-6
   - Follow the exact implementation from the research doc

5. **`compute_execution_reward(generated_sql, gold_sql, gold_records, mem_conn)`**
   - Graded reward per user decision (NOT binary):
     - +1.0: `gen_records == gold_records` (exact match)
     - +0.5: partial overlap (Jaccard similarity of record sets, > 0 overlap)
     - -0.5: executes but wrong records (no overlap)
     - -1.0: SQL error (execution returns None)
   - Import `_execute_sql` from `part1.dpo_data`
   - Handle gold_records=None as 0.0 (skip -- gold SQL failed)

6. **`compute_kl_penalty(current_log_probs, ref_log_probs)`**
   - Simple KL: `(current_log_probs - ref_log_probs).mean()`
   - Returns scalar penalty (to be multiplied by kl_beta in the training loop)

All functions should operate on torch tensors and be differentiable where needed (losses). Reward and advantage functions are non-differentiable (no grad needed).

Add module docstring explaining the relationship between GRPO and CISPO.
