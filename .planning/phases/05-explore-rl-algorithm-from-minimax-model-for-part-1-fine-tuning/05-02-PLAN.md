**Function `sample_group_completions(model, vocab, tokenizer, nl_texts, gold_sql_list, gold_records_list, mem_conn, cfg, device)`:**

This function handles the SAMPLE + REWARD phases of the GRPO/CISPO loop.

Steps:
1. Prepend schema string to NL texts (using `_load_schema_string(mode=cfg.schema_mode)`)
2. Tokenize with `_TOKENIZER` (padding=True, truncation=True, return_tensors="pt")
3. Move to device
4. Expand input_ids and attention_mask by `repeat_interleave(cfg.group_size, dim=0)` for group sampling
5. Call `model.model.generate()` (inner HF model) with:
   - `do_sample=True`, `temperature=cfg.sampling_temperature`, `top_k=cfg.sampling_top_k`
   - `num_return_sequences=1` (already expanded)
   - `decoder_start_token_id=32099` (same as DPO data generation)
   - `prefix_allowed_tokens_fn=vocab.get_prefix_allowed_tokens_fn()`
   - `max_new_tokens=cfg.max_new_tokens`
   - Wrap in `torch.inference_mode()` and `_amp_context(cfg.use_amp, device)`
6. Decode outputs with `tokenizer.batch_decode(outputs, skip_special_tokens=True)`
7. Apply same post-processing as DPO: `re.sub(r'(?<=[a-zA-Z0-9_)])\s*,\s*', ' , ', s)` for each decoded string
8. Compute execution rewards for each of the B*G completions using `compute_execution_reward(gen_sql, gold_sql, gold_records, mem_conn)`
   - gold_sql and gold_records are repeated to match the B*G flat structure
9. Return:
   - `completions`: list of decoded SQL strings (B*G,)
   - `rewards`: torch.Tensor of shape (B*G,)
   - `generated_ids`: the raw token IDs from generate (B*G, T_gen) for log prob recomputation

**Important design note:** Do NOT use `.generate()` scores for log prob computation. Recompute log probs via `compute_restricted_log_probs()` to avoid precision mismatches (per research pitfall #3).

**Function `compute_old_log_probs(model, encoder_input, encoder_mask, generated_ids, cfg, device, per_token=False)`:**

Compute log probabilities of the generated sequences under the old/reference policy.

Steps:
1. Build decoder_input: prepend BOS token (`_BOS_ID`) to generated_ids[:, :-1]
2. Set decoder_targets = generated_ids
3. Call `compute_restricted_log_probs(model, encoder_input, encoder_mask, decoder_input, decoder_targets, pad_idx=PAD_IDX, per_token=per_token)` with `torch.no_grad()` and `_amp_context()`
4. Return log probs (detached)

For LoRA reference: use `model.model.disable_adapter_layers()` before, `model.model.enable_adapter_layers()` after (same pattern as `dpo_train_step_lora`).

**Function `grpo_train_step(policy_model, batch_nl, batch_gold_sql, batch_gold_records, vocab, mem_conn, optimizer, cfg, device, grad_norm_ema, global_step)`:**

Single GRPO/CISPO training step combining all phases. This is the heart of the training loop.

Steps:
1. **SAMPLE PHASE:** Call `sample_group_completions()` to get completions, rewards, generated_ids.
2. **ADVANTAGE PHASE:** Call `compute_group_advantages(rewards, cfg.group_size, cfg.use_std_normalization)`.
   - If `cfg.skip_dead_groups` is True: create a mask of non-dead samples (where group std > 1e-6). Zero out advantages for dead groups so they contribute zero gradient.
3. **Prepare encoder inputs** for log prob computation: re-tokenize the batch_nl with schema, move to device, expand by group_size.
4. **OLD LOG PROB PHASE:** Call `compute_old_log_probs()` with the policy model (LoRA disabled for reference) or frozen model.
   - For CISPO with per-token: use `per_token=True`
   - For GRPO: use `per_token=False` (sequence-level)
5. **UPDATE PHASE (gradients enabled):**
   - Build decoder_input and decoder_targets from generated_ids (same as compute_old_log_probs)
   - Compute current policy log probs via `compute_restricted_log_probs()` with AMP context
   - Select loss function based on `cfg.rl_algorithm`:
     - "grpo": `grpo_loss(current_logps, old_logps, advantages, cfg.epsilon)`
     - "cispo": If per_token log probs available, use `cispo_loss_per_token()`. Otherwise fall back to sequence-level `cispo_loss()`.
   - Add KL penalty if `cfg.kl_beta > 0`: `loss = loss + cfg.kl_beta * kl_penalty`
   - `optimizer.zero_grad()`
   - `loss.backward()`
   - Gradient clipping: `torch.nn.utils.clip_grad_norm_(policy_model.parameters(), cfg.grad_clip_norm)`
   - **Gradient norm spike detection:** Compare current grad_norm to EMA. If `grad_norm > cfg.max_grad_norm_spike_factor * grad_norm_ema`, skip this optimizer step (zero gradients, don't step). Log the spike.
   - Update EMA: `grad_norm_ema = cfg.grad_norm_ema_decay * grad_norm_ema + (1 - cfg.grad_norm_ema_decay) * grad_norm`
   - `optimizer.step()`
6. **Return** metrics dict: `loss, grad_norm, mean_reward, zero_std_frac, clip_frac, mean_ratio, kl_penalty, grad_spike_skipped, grad_norm_ema` and the updated `grad_norm_ema`.

**Function `grpo_train(cfg, policy_model, train_nl, train_sql, train_gold_records, dev_loader, vocab, mem_conn, optimizer, run_dir)`:**

Main training loop. Follow exactly the `dpo_train()` pattern from `dpo_train.py`:

1. Setup: ckpt_dir, best_val, best_metrics, epochs_since_improvement, train_start, gt paths.
2. **Epoch loop** (cfg.num_epochs iterations):
   a. Shuffle training data indices each epoch (on-policy: fresh samples every epoch).
   b. **Mini-batch loop:** Iterate over training data in chunks of `cfg.batch_size` queries.
      - Extract batch_nl, batch_gold_sql, batch_gold_records
      - Call `grpo_train_step()` for this mini-batch
      - Accumulate epoch metrics
      - Log batch-level metrics to W&B (step=global_step)
   c. Compute epoch averages.
   d. Print epoch summary line.
   e. **Eval** (every `cfg.eval_every_n_epochs` epochs): Same eval pattern as `dpo_train`:
      - `eval_epoch_gpu()` for GPU inference
      - `eval_epoch_sql()` for SQL execution metrics
      - Log all metrics to W&B
      - Checkpoint if improved (save_model with best=True)
   f. **Early stopping:** Same as dpo_train (patience_epochs check).
   g. **Wall clock budget:** Same check.
   h. **Graceful stop:** Same `stop_requested()` check.
3. Print final summary.
4. Return best_val.

**W&B Metric Namespacing:**
- Standard: `epoch, train_loss, record_f1, record_em, sql_em, error_rate, gradient_norm, lr`
- RL-specific: `rl/mean_reward, rl/zero_std_frac, rl/clip_frac, rl/mean_ratio, rl/kl_penalty, rl/grad_spikes_skipped`
- Batch: `batch/loss, batch/gradient_norm, batch/mean_reward, batch/clip_frac`
- Timing: `timing/epoch_seconds, timing/wall_clock_seconds`
- Tracking: `tracking/best_record_f1, tracking/epochs_since_improvement`
