Key insight: Training epochs never block waiting for SQL. After Eval GPU generates SQL strings via beam search, SQL execution is dispatched to a background thread. Training resumes immediately. Results are collected non-blockingly (future.done()) at the start of each epoch. Only the final epoch blocks to drain all pending results.

Config: eval_every_n_epochs=3 means SQL from epoch 2 overlaps with training of epochs 3, 4, 5.
